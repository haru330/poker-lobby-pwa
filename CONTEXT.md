# Poker Lobby PWA

A peer-to-peer (WebRTC) lobby and poker game framework where one device acts as the authoritative Host, holding all game state, and other devices join as Guests over the same wifi.

## Language

**Host**:
The device that owns the authoritative game state, performs all game logic/calculations, and persists state to its local store.
_Avoid_: Server, dealer (dealer is a poker role, not a network role)

**Guest**:
A device connected to the Host that sends actions and receives broadcast state updates.
_Avoid_: Client, peer (peer is a WebRTC term, not a role)

**Room Code**:
The 4-letter code identifying a lobby, used as the PeerJS ID for the Host.
_Avoid_: Passcode, lobby ID

**Session Token**:
A random ID generated by a Guest on first join and stored in their local storage, used by the Host to match a reconnecting Guest back to their existing seat.
_Avoid_: Player ID, user ID

**Table**:
A lobby's set of seats, capped at 8 players (standard full-ring Texas Hold'em). Joins beyond this cap are rejected as "lobby full".
_Avoid_: Lobby (Lobby is the pre-game screen; Table is the seat capacity concept)

**Street**:
A betting round tied to a stage of community-card reveal: Preflop (0 community cards), Flop (3), Turn (4), River (5). Each street ends when all non-folded, non-all-in players' contributions are equal.
_Avoid_: Round, phase (Phase is the app-level screen state)

**Hand Category**:
The evaluator's classification of a player's best 5-card hand (High Card through Royal Flush), with full tiebreak ranks for comparing same-category hands.
_Avoid_: Hand rank, hand score

**Hand Strength Bar**:
A visual bar (0-100%, colored, with a verdict like "decent"/"strong"/"very strong") shown to a player for their current hand. Preflop (2 cards, no Hand Category yet distinguishes AA from 72o), it's derived from hole-card quality (pair/suited/connectedness, via a Chen Formula variant). From the Flop onward, it's derived from the player's current Hand Category (best-of-N for N=5,6,7 community+hole cards), plus optional Board Texture Concerns.
_Avoid_: Equity, odds (this is category/heuristic-based, not probability-based)

**Board Texture Concern**:
A predetermined caution note (e.g. "the board allows a possible flush") shown alongside the Hand Strength Bar when community cards make a stronger hand possible for an opponent. Based only on board composition, never on opponents' bets or hidden cards.
_Avoid_: Suggestion, advice

**Dealer Button**:
A marker on one seat that rotates clockwise to the next connected, non-folded player at the start of each hand; determines who posts Small/Big Blind and who acts first.
_Avoid_: Dealer (Dealer is a role marker, not a person — the Host deals)

**Big Blind**:
The forced bet posted by the player two seats clockwise of the Dealer Button at the start of each hand, set by the Host as the table's stakes unit.
_Avoid_: Ante (this framework has no separate per-player ante)

**Small Blind**:
The forced bet posted by the player one seat clockwise of the Dealer Button, equal to half the Big Blind (rounded down).
_Avoid_: -

**Action Log**:
A broadcast record of each confirmed action (who acted, what they did) plus whose turn is next, shown to all players.
_Avoid_: Activity feed, history

**Pending Action**:
An action (raise/call/check/fold) sent by a Guest, shown to the Host as an editable proposal before being confirmed and applied to game state.
_Avoid_: Action request, draft action

**Host Migration**:
The process by which, when the current Host disconnects, the longest-connected remaining Guest becomes the new Host using its locally synced copy of the full game state.
_Avoid_: Failover, handoff

**Combo Cards**:
The subset of a player's best-5 cards that defines the Hand Category itself (e.g. both cards of a Pair, all three of Three of a Kind, all 5 cards of a Straight/Flush/Full House). Highlighted in green, distinct from the remaining best-5 cards (kickers), which are highlighted in white.
_Avoid_: Highlighted cards (too generic — Combo Cards are a subset of the highlighted best-5)

**Tiebreak Note**:
A short description (e.g. "K kicker") shown at Showdown explaining which rank decided a pot between two players who shared the same Hand Category.
_Avoid_: Kicker (Tiebreak Note covers the general case, not just literal kickers)
